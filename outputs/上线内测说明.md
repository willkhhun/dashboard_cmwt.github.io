# 潮鸣外滩数字营销看板｜快速上线内测说明

## 1. 内测版能力

当前版本已经支持两种模式：

- 本机模式：不配置云端时，继续保存在当前浏览器。
- 云端内测模式：配置 Supabase 后，团队成员访问同一个网址，共用同一份看板数据。

内测版采用“单工作区共享数据”，适合先让负责人、主播、运营一起试用。后续正式版可以再拆分账号权限、操作日志和更细的数据表。

## 2. 创建 Supabase 数据库

1. 打开 Supabase，新建一个项目。
2. 进入 SQL Editor。
3. 复制 `supabase-setup.sql` 的内容并运行。

默认工作区 ID 是：

```text
chaoming-waitan-beta
```

如果要修改工作区 ID，需要同步修改：

- `supabase-setup.sql` 里的 `chaoming-waitan-beta`
- `cloud-config.js` 里的 `workspaceId`

## 3. 填写云端配置

打开 `cloud-config.js`，把 `enabled` 改成 `true`，并填写 Supabase 项目的 URL 和 anon key：

```js
window.CHAOMING_CLOUD_CONFIG = {
  enabled: true,
  provider: "supabase",
  supabaseUrl: "你的 Supabase Project URL",
  supabaseAnonKey: "你的 Supabase anon public key",
  workspaceId: "chaoming-waitan-beta"
};
```

anon key 是前端公开 key，适合内测快速使用。正式版建议增加登录、角色权限和更严格的数据库策略。

## 4. 部署网页

把 `outputs` 文件夹作为静态网站部署即可。可选方式：

- Vercel
- Netlify
- 阿里云 OSS 静态网站
- 腾讯云 COS 静态网站
- 公司内网静态服务器

部署后，团队成员打开同一个网址即可访问。

## 5. 使用方式

页面左侧会显示同步状态：

- 本机模式：没有启用云端配置。
- 云端连接中：正在连接 Supabase。
- 云端已同步：已读取云端数据。
- 云端已保存：本次修改已保存到云端。
- 云端读取/保存失败：需要检查 Supabase 配置、网络或数据库策略。

内测版每次修改会自动保存到云端，并每 15 秒拉取一次其他成员的更新。

## 6. 内测限制

- 暂时没有账号登录。
- 暂时没有分角色权限。
- 多人同时编辑同一项时，以最后保存者为准。
- 手机号等敏感数据在正式上线前建议增加登录权限和访问控制。

建议先用 3-5 人小范围内测，确认业务流程顺手后，再升级正式多人权限版。
